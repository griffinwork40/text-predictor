TextPredictor is a system-wide text predictor for Apple Silicon. It watches what you type, waits 350ms after you stop, then suggests what comes next — inline, inside whatever app has focus, without a character leaving your machine.

What makes it different
01

No server. The model runs on your chip.

Qwen3-1.7B-4bit runs in-process via Apple's MLX framework, compiled to Metal shaders that execute on the Neural Engine. After the JIT warmup on first launch, steady-state latency is around 30ms. Nothing is sent anywhere. The inference actor is an actor Inference that keeps MLX work off the main thread and supports mid-generation cancellation if you start typing again.

02

No plugin. The macOS Accessibility API does the work.

Ghost text is injected via kAXSuggestedValuesAttribute — the same Accessibility API that powers system autocorrect — directly into the focused element of any app. Notes, Mail, Slack, Terminal, Xcode, wherever. No per-app integration, no browser extension, no keyboard remapping daemon.

03

No subscription. Build it once, it's yours.

MIT license. The model weights are a one-time ~2GB download from HuggingFace. After that it runs indefinitely at zero marginal cost. There's no notarized .app yet — you build from source with Xcode 26 and run the binary from its product directory so it can find the colocated Metal bundle.

How it works

An AX observer watches the focused text element for AXValueChanged events. When you stop typing for 350ms, it reads the current text up to the caret, trims to the last 3 words for context, and queues an inference request.

Inference runs on a dedicated Swift actor backed by mlx-swift-lm. A new keystroke or new Ctrl+Space cancels any in-flight generation. When a completion arrives, it's injected as kAXSuggestedValuesAttribute on the focused element. If the target app doesn't support that attribute — which happens — a floating NSPanel appears instead.

Tab accepts. Acceptance is synthesized via CGEvent.keyboardSetUnicodeString, which flows through the app's normal input path. Undo history, autocorrect, and rich text formatting all work correctly because the text goes through the same path as real keystrokes. Esc, or any other key, dismisses the suggestion.

The only visible UI is a ✦ status item. There is no Dock icon, no window, no preferences pane.

Get started

Clone the repo, build, and run. The model downloads on first inference.

$ git clone https://github.com/griffinwork40/text-predictor
$ cd text-predictor
$ ./run.sh

# First run: grants Accessibility permission, downloads
# Qwen3-1.7B-4bit weights (~2GB), JIT-compiles Metal shaders.
# Subsequent launches are fast.

After launch, a ✦ appears in your menu bar. Start typing in any app and pause for 350ms — or press Ctrl+Space to trigger manually.

Requirements
macOS 14 Sonoma or later
Hardware Apple Silicon — M1 or later
Xcode 26 Beta with Metal Toolchain
xcodebuild -downloadComponent MetalToolchain

~/.cache/huggingface/hub/models--mlx-community--Qwen3-1.7B-4bit/
Permissions Accessibility — required for AX API access and synthesized input

This is an M1A vertical-slice spike — the milestone that proves the full pipeline works end-to-end. AX read → inline injection → MLX inference → Tab accept. The model is real, the inference is real, the ghost text is real. What's missing is the packaging: no notarized app, no auto-updater, no settings UI. That's M1B through M1D.
